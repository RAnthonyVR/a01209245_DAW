Preguntas

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

La función session_unset() libera todas las variables asignadas de sesión. [1] Mientras Que sesión_destroy() destruye todos los datos asociados con la sesión actual. Pero no elimina las variables asociadas con la sesión. [2]

¿Cuál es la diferencia entre una variable de sesión y una cookie?

Una variable de sesión se almacena del lado del servidor y solo se mantiene durante la sesión de un usuario mientras que una variable cookie se almacena del lado del cliente. Es por ello por lo que las cookies tienen un tiempo de vida mayor que las sesiones. Ambos se utilizan para generar persistencia entre los datos. [3]

¿Qué es CSRF y cómo puede prevenirse?

La vulnerabilidad Cross Site Request Forgery. Obliga al navegador web de una víctima, validado en algún servicio o sesión a enviar una petición a una aplicación web vulnerable. Se puede preevenir configurando el navegador, cerrar las sesiones. [4]
