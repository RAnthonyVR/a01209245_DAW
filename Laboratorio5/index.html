Preguntas

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    Cabe mencionar JavaScript y Java no están relacionados en sí, JavaScript decidió incluir al nombre el “Java” porque cuando surgió JavaScript, Java era muy popular.
    Porque al momento de enviar datos al servidor y previamente haberlos validado en JavaScript, entonces nos aseguramos de que todos los datos del servidor son válidos. Además, es muy común que los usuarios se equivoquen e ingresen datos inválidos. Al hacer la validación desde el lado del cliente y no desde el lado del servidor se puede ahorrar el tiempo de respuesta a algún cliente, haciéndolo de manera inmediata. Esto ahorra el tener que validar de alguna forma toda la información que contiene el servidor. [1]
    Entre los distintos beneficios de poner esto en práctica se encuentran: Asegurar que la información sea de un tipo de dato específico, correos electrónicos validos y el asegurar que alguna información necesaria sea ingresada por el usuario (por medio de los campos obligatorios). También conocidos como valores nulos. [2]
  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    Puedes desabilitar Javascript en el buscador, para ello solo es necesario ir a las configuraciones especificas del buscador. Esto desabilita completamente Javascript.
  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    Porque no todas las personas o usuarios son consientes de esta opción. Además de ello, la validación desde el lado del cliente además de tener beneficios de seguridad le ayuda al cliente a tener una respuesta inmediata ante una solicitud. Aun así existen validaciones desde el lado del servidor, pero tener un filtro desde el lado del cliente siempre es un buena práctica
