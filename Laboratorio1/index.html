Investigación

Conceptos

  • URL (Uniform Resource Locator) Es la dirección de una página HTML o recurso en Internet, cuenta principalmente con el nombre del recurso y el identificador de protocolo (por ejemplo HTTP). [1]
  • MIME (Multipurpose Internet Mail Extensions) Es un estándar del Internet para compartir archivos de datos como imágenes, audio, video, texto, etc… en un mensaje o email. [2]
  • Métodos y códigos de errores http (Hypertext Transfer Protocol) Los métodos permiten la lectura, creación y actualización de un recurso especifico. [3] Los errores son la respuesta de un servidor a la petición de algún cliente, forman parte del los códigos de estado. Los cuales empiezan con un primer digito estandarizado que se clasifica en 5 tipos: 1xx Informativos: indican continuación de proceso. 2xx: Éxito. 3xx: Re direccionamiento. 4xx: Error del cliente. 5xx: Error del servidor. [4]
  • XML (Extensible markup language) Según Dykes, L. en el libro "XML for Dummies" es lenguaje marcado de texto. Sirve como herramienta para almacenar y compartir datos en texto simple. A diferencia de HTML, las etiquetas que se usan no son predefinidas. No depende del software ni de hardware. [5]
  • XHTML Es HTML rediseñado como XML, es decir, es un lenguaje donde se marca de manera correcta todo el documento con el objetivo de funcionar en la mayoría de los buscadores y dispositivos sin problema alguno. Contiene estructuras obligatorias para cumplir este propósito. [6]
  • HTML5 Es una nueva versión del lenguaje de desarrollo web HTML, su objetivo es mejorar la experiencia de desarrolladores y usuarios web. Introduce nuevos componentes y atributos. [7]
  • JS (JavaScript) Es un lenguaje de programación y se ha utilizado para las nuevas aplicaciones web modernas, mejorando la interacción de los usuarios. No tiene casi nada que ver con Java. [8]
  • Php (Hypertext processor) Es uno de los lenguajes de programación más populares de desarrollo web, puede trabajar en conjunto con paginas de HTML. Permite la creación de paginas web dinámicas e interactivas como el establecer una conexión con alguna base de datos y obtener información de esta. [9]
  • jsp (Java Server Pages) Es una tecnología de programación que permite crear de manera dinámica y estática contenido Web. [10]
  • Ajax (Asynchronous JavaScript And XML) No es un lenguaje de programación, permite mantener una página actualizada sin tener que recargar una página e interactuar con el servidor. [11]

Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web? El Internet es una red de computadoras (red de redes) , fue conceptualizada en 1969 mientras que la World Wide Web se utiliza para acceder al internet. Esta constituido por páginas web que utilizan el protocolo HTTP utilizado en todo el mundo, el cual es un lenguaje que permite la comunicación en el internet. Esto permite en conjunto que personas de alrededor del mundo tengan acceso a un gran contenido de información. [12]
  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? El propósito de estos métodos es el establecimiento de comando que permitan el manejo de comunicación en la web. GET: obtiene información de un servidor proporcionando un URL. HEAD: Obtiene los headers y la línea de estado de un servidor sin información o cuerpo. POST: Crea un recurso. PUT: Reemplaza el contenido anterior con uno nuevo. PATCH: Modifica parcialmente un recurso. DELETE: Suprime las representaciones de un recurso en especifico. Otros: CONNECT: Establece una conexión al servidor dado un URL. OPTIONS: Describe las opciones de comunicación. TRACE: Envía un mensaje y se recibe el mensaje de solicitud del servidor para hacer una prueba. [13]
  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo, cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? El método POST porque además de que permite enviar datos al servidor, lo hace de manera segura y es utilizado para datos privados. [14]
  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? El método GET porque lleva un mensaje al cliente, a través de un URL enviado por el mismo. [15]
  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? Significa OK, que hubo éxito en la solicitud, no ocurrió algún error. El cliente pudo ver la página solicitada sin problemas. [16]
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? El error 404 “Not Found” No es responsabilidad del desarrollador corregirlo porque implica que el cliente ingresó incorrectamente el URL o la página se cambió de sitio web. El cliente pudo comunicarse con el servidor, pero este no pudo encontrar lo solicitado. [17]
  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? Si es responsabilidad de un desarrollador corregirlo porque significa que hay un error desconocido en el servidor de la página. Al ser un error interno del servidor, significa generalmente que hay un error en la programación de la página. [18]
  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Significa que ese atributo antes usado en HTML4, ya no se encuentra disponible en HTML5 porque se utilizan en CSS o han sido eliminados por completo. Algunos elementos desaprobados: link, rev. Shape, name, align, entre otros. [19]
  • ¿Cuáles son las diferencias principales entre HTML4 y HTML5? HTML5 es una versión avanzada de HTML4. HTML5 es más simple a comparación de HTML4. HTML5 contiene mejor manejo de errores y consistencia en documentos. HTML5 contiene nuevas etiquetas como video y audio. Aunque Html4 tiene mayor compatibilidad a comparación de html5. [20]
  • ¿Qué componentes de estructura y estilo tiene una tabla? Estructura: table: Define una tabla. th:Define una celda de encabezado en la tabla. tr: Define una fila en una columna. td: Define una sola celda en una tabla. body y tfoot. Los components de estilo que se pueden utilizar son: border, border-collapse, padding, text-align, border-spacing, colspan, rowspan y id. [21]
  • ¿Cuáles son los principales controles de una forma HTML5? form: Define una forma HTML que ingresa datos del usuario. input: Establece un control de ingreso de datos. textarea: Establece una área de texto para ingreso de datos. label: Define una etiqueta para un elemento de entrada. [22]
  • ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/ (Al responder la pregunta recuerda poner el navegador que utilizas) Utilizo Safari 12.0.2 on macOS Mojave 10.14 y obtuve 456 de 555 puntos. [23]
  • ¿Cuál es el ciclo de vida de los sistemas de información? Según Manuela Aparicio, el ciclo de vida de un sistema de información tiene un límite de vida que puede llegar a extender, pero estos mueren. Se puede llegar a dividir el ciclo en: Lanzamiento: Se identifican las necesidades y se proponen soluciones. (Análisis de requisitos) Desarrollo: comienza a implementarse y expandirse el sistema. Madurez: Se hace un mantenimiento a las aplicaciones, usuarios y auditorias. Declive: Última fase del sistema, el sistema debe de ser remplazado o renovado. [24]
  • ¿Cuál es el ciclo de desarrollo de sistemas de información? Investigación preliminar: Breve investigación sobre las actividades de una organización, identificación de problemas y objetivos del proyecto. Análisis: Se analiza a detalle la gestión de información de una organización. Para ello se pueden utilizar entrevistas, cuestionarios, organigramas, etc… Diseño del sistema: Se diseña un sistema con el objetivo de solucionar los problemas mencionados. Desarrollo y documentación del sistema: Con base a la documentación del sistema se crea un diseño. Implantación del sistema de información y conocimientos. Mantenimiento del sistema: adecuar el sistema a futuros cambios o nuevas necesidades. [25]
