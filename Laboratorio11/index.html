Preguntas


  • ¿Por qué es una buena práctica separar el controlador de la vista?
    Separar la interfaz de la lógica o funcionamiento además de hacer agregarle al sistema mantenibilidad y escalabilidad, es decir agregarle y modificar las funcionalidades de forma sencilla. Permite que un cambio de funcionabilidad no tenga que modificar o afectar a múltiples secciones de código además de que se puede optar por cambiar lenguajes o lógica del sistema de manera independiente de la vista, delegando una sola responsabilidad por cada código. [1]
  • Aparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?
    • $_SERVER: Contiene información del ambiente de ejecución y del servidor. ( headers, paths y ubicaciones de scripts.)
    • $_FILES: Contiene variables para subir archivos de http al script. Utiliza el método POST.
    • $_REQUEST: Contiene variables de petición en http. (Por default tiene como conteniedo a $_GET, $_POST Y $_COOKIE).
    • $_SESSION: : Contiene variables de las sesione disponibles en el script actual.
    • $_SESSION: Contiene variables de entorno pasadas al script.
    • $_SESSION: Contiene cookies de http.(variables paras al script)

    [2]

  • ¿Qué es XSS y cómo se puede prevenir?
    Es una vulnerabilidad conocida como Cross-site Scripting en la que los atacantes explotan la confianza que un usuario tiene en un sitio en particular y volver peligroso para sus usuarios a un sitio legítimo. Para evitarlo se puede contar con una solución de seguridad instalada y actualizada. Los usuarios lo pueden prevenir viendo el enlace URL y asegurándose que sea seguro. Los desarrolladores deben prevenir los ataques XSS al escapar de su output usando htmlspecialchars () o htmlentities (). Ambas funciones de PHP convierten los caracteres problemáticos en entidades HTML, lo que hace que el código inyectado se imprima de manera inofensiva y no se renderice. [4]
